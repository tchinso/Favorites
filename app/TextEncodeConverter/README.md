# Text Encoding Lab

브라우저에서 파일을 외부로 전송하지 않고 문자 인코딩을 감지하고 변환하는 GitHub Pages 앱입니다.

## 핵심 동작

- BOM을 최우선으로 확인하고, BOM이 없으면 인코딩별 바이트 문법을 엄격히 검증합니다.
- 유효한 후보만 한국어·일본어·중국어·라틴 문자 분포와 파일 형식 단서로 비교합니다.
- CP949 확장 영역을 검사해 EUC-KR과 구분하고, 공통 영역뿐인 경우에는 구별 불가능하다고 명시합니다.
- 변환 뒤 다시 디코드하여 원문과 코드 포인트 단위로 비교합니다. 손실은 기본적으로 다운로드 전에 차단됩니다.
- UTF-8/16/32의 BOM 포함 여부를 출력 옵션에서 명시적으로 선택합니다.

## 지원 인코딩

- Unicode: UTF-8, UTF-16 LE/BE, UTF-32 LE/BE (각 BOM 정책 포함)
- 한국어: CP949/UHC, EUC-KR/KS X 1001, ISO-2022-KR
- 일본어: CP932/Windows-31J/Shift_JIS, EUC-JP, ISO-2022-JP
- 중국어: GB18030, GBK, Big5
- 서유럽/기타: Windows-1252, ISO-8859-1, ISO-8859-15, ASCII

`JOHAB/CP1361`은 현재 번들 디코더에 신뢰할 수 있는 매핑 테이블이 없어 이름만 지원하는 척하지 않습니다. 파일이 ASCII 범위만 사용하거나 EUC-KR과 CP949의 공통 영역만 사용하는 경우 원 인코딩은 원리적으로 확정할 수 없으며 앱도 낮은 신뢰도로 표시합니다.

## 실행과 검증

정적 파일이므로 `index.html`을 HTTP 서버로 열면 됩니다. GitHub Pages에서는 저장소 루트(또는 Pages로 지정한 폴더)를 그대로 배포합니다.

```bash
npm test
```

테스트는 Unicode BOM, 한국어·일본어 레거시 왕복 변환, 자동 감지, CP949/EUC-KR 구분, 잘못된 바이트 거부, 손실 감지, 줄바꿈 보존과 잘린 대용량 표본을 검사합니다.
